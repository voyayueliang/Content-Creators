# 人机协作边界与内容判断方法

## 一句话

内容创作者使用 AI 前的底层方法卡：先分清 AI 可以代劳的劳动、必须留给人的判断、需要保护的关系边界和必须校对的事实。

## 适合谁

- 深度内容创作者
- 社区运营者和活动组织者
- 播客、访谈、直播内容整理者
- 用 AI 写商业合作稿、人物稿、活动稿的人
- 正在搭建个人内容工作台的创作者

## 什么时候用

- 开始任何内容工作流前。
- AI 初稿看起来很顺，但你觉得“不对劲”时。
- 文章涉及合作方、人物、未成年人、客户、活动现场或公开授权时。
- 内容要进入公众号、作品集、客户交付或公开传播前。
- 你想让 AI 帮忙，但不想让它替你做价值判断时。

## 月亮的方法

月亮把 AI 当作第二张工作台。AI 可以把材料摊开、提取结构、生成候选、比较版本、提醒事实缺口和回收反馈；人的工作是判断这件事值不值得写、哪些材料可以公开、哪些表达像自己、哪些关系需要保护，以及最终是否发布。

这套方法的核心是：自动化桌面劳动，保留人的判断、现场、关系、证据和最终责任。

## 输入

- 原始材料：文档、录音、逐字稿、策划案、图片、链接、草稿或项目经历。
- 发布场景：公众号、作品集、客户交付、内部整理、公开演讲或社群分发。
- 读者对象：普通读者、编辑、合作方、活动参与者、潜在客户或未来伙伴。
- 风险提醒：事实错误、AI 味、公开边界、授权、关系分寸、语气错位。

## 输出

- 人机协作边界卡。
- AI 可代劳事项清单。
- 人必须判断事项清单。
- 事实、证据、关系和公开边界清单。
- 下一步建议：继续写、补材料、先调研、先确认边界或暂不公开。

## 使用步骤

1. 先描述当前内容任务和使用场景。
2. 列出原始材料和已经确定的信息。
3. 标出最担心的风险：事实、声音、关系、授权或商业口径。
4. 让 AI 先做边界卡，不直接写正文。
5. 人确认边界卡后，再进入对应工作流。
6. 成稿后再过事实校对门和声音守门。

## 人机边界

AI 可以：

- 整理材料和时间线。
- 提取人物、场景、问题、证据和缺口。
- 生成结构候选和表达候选。
- 标出可能的事实风险、授权风险和 AI 味。
- 把一次反馈回流成规则。

人必须保留：

- 内容角度和价值判断。
- 发布主体的位置。
- 公开边界和关系判断。
- 哪句话是否像自己或像发布主体。
- 是否交付、发布、署名、作为案例。

## 质量门

| 门 | 要问的问题 |
| --- | --- |
| 判断门 | 这篇内容真正要回答什么问题？ |
| 证据门 | 哪些说法有材料支撑，哪些需要弱化？ |
| 边界门 | 是否涉及隐私、授权、合作方口径或敏感关系？ |
| 声音门 | 有没有空泛总结、过度顺滑、万能转折和不像人的表达？ |
| 责任门 | 最终是否发布、如何署名、谁承担公开表达责任？ |

## 启动句

```text
打开人机协作边界与内容判断方法。

我现在要处理的内容是：[活动稿 / 人物稿 / 口述材料 / 演讲 PPT / 作品集 / 商业合作稿 / 其他]
原始材料在这里：[粘贴材料、链接或文件路径]
发布或使用场景：[公众号 / 作品集 / 客户交付 / 内部整理 / 公开分享 / 不确定]
我最担心的问题：[事实错误 / 太像 AI / 边界不清 / 判断被写偏 / 关系风险 / 其他]

请先不要直接写正文，先帮我做一张人机协作边界卡。
```
