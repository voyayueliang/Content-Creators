月亮的 AI 工作台

为内容创作者、深度社区活动组织者等打造的个性化内容工作流。我们洞察场景需求，设定人机协作的边界，在一次次交付中积累判断力。这里不只是 human in the loop，更是 humanity in the loop。

3 分钟开始

  1. 1选一个最接近的内容场景。
  2. 2把材料、发布目的和不能公开的边界放进去。
  3. 3先让工作流做材料地图和缺口检查。
  4. 4再写稿，最后过事实校对和去 AI 味。

适合谁

这个工作台不是给空白写作准备的，它更适合已经有材料、但缺少整理、判断和交付流程的人。

社区运营 / 活动主办方

需要把活动资料、嘉宾议程和合作方口径变成能发布的传播稿。

播客 / 直播主理人

有录音、回放或逐字稿，想做二次分发，又不想只得到一份摘要。

内容实习生 / 编辑

要承接小型内容需求，但又必须控制事实、授权、口径和 AI 味风险。

独立创作者 / 研究者

想把公开资料、观察和项目经验，沉淀成可复用的内容资产。

四个内容生产场景

先解决一个具体任务，再慢慢沉淀成自己的内容系统。每个入口都保留部署包和启动句，但页面默认只展示判断信息。

活动预告 / 分论坛介绍 / 商业合作稿

社区活动内容传播工作台

把策划案、会议纪要、议程和合作方口径，整理成可审核的活动传播稿。

应用场景

社区活动预告、论坛介绍、合作方发布稿、商业合作内容交付。

痛点

信息很多，但传播稿容易变成资料堆砌；商业合作又不能出错。

怎么用
  1. 准备策划案、嘉宾议程和会议纪要。
  2. 补充合作方价值和不能公开的信息。
  3. 先做事实缺口检查，再进入草稿。
输入 / 输出样例

输入：活动策划案、议程、嘉宾信息、合作方口径。
输出：公众号草稿、标题备选、事实待确认清单。

会后报道 / 活动复盘 / 社区记忆

活动复盘转公众号

把照片、群聊反馈、口述复盘、公开报道和伙伴记录，整理成会后传播包。

应用场景

活动结束后 24-72 小时内的报道、复盘、伙伴 proof 和社区记忆整理。

痛点

材料很散，热度很快过去；不及时整理，现场价值和伙伴反馈都会流失。

怎么用
  1. 准备复盘口述、照片说明和公开报道链接。
  2. 标注哪些图片、名字和评价可以公开。
  3. 先生成素材地图，再判断写报道还是传播包。
输入 / 输出样例

输入：复盘录音、现场照片、群聊反馈、伙伴记录。
输出：会后报道、图片占位说明、社区记忆素材包。

播客 / 录音 / 视频回放 / 直播

播客音视频转公众号

把音视频内容转成公众号文章、对谈稿、短观察或可继续复用的素材资产。

应用场景

播客、访谈、直播回放、圆桌逐字稿、长视频内容再分发。

痛点

音视频信息密度高；只做总结会丢掉说话人的语气、关系和现场感。

怎么用
  1. 准备逐字稿、摘要或关键片段。
  2. 说明说话人身份、发布平台和授权边界。
  3. 先判断内容形态，再决定长文、对谈稿还是短观察。
输入 / 输出样例

输入：播客逐字稿、直播摘要、嘉宾信息、公开边界。
输出：公众号文章、对谈稿、短观察或素材卡。

公众人物 / 创始人 / Builder Profile

公众人物深度报道工作台

基于公开访谈、播客、演讲、文章和社媒资料，写有事实约束的人物深度报道。

应用场景

公众人物、创始人、开发者、创作者、行业人物的资料研究型长文。

痛点

资料很多，但容易写成百科、PR 稿或主观判断；需要先分清事实、引用、推断和边界。

怎么用
  1. 给出人物姓名和公开材料链接。
  2. 说明发布渠道、读者和不能推断的边界。
  3. 先做资料查看台和文章角度选择，再写逐节草稿。
输入 / 输出样例

输入：人物姓名、公开访谈、播客、演讲、文章链接。
输出：资料查看台、文章角度、逐节人物稿和引用核查表。

Final Gate

去 AI 味与声音守门

发布前最后一轮：把 AI 初稿里过度顺滑、空泛判断、硬凑转折和不像人的表达挑出来，改回具体、克制、有边界的人的声音。

查看启动句 
打开去 AI 味与声音守门工作流。

当前稿在这里：[粘贴稿件]
原始材料在这里：[粘贴材料或文件路径]

请标出不像人的地方、为什么不稳，并给出更像我的改法。
同时检查是否有空泛判断、硬凑转折、过度顺滑和没有材料支撑的漂亮话。
应用场景

公众号发布前、商业合作稿审核、AI 初稿改稿、人物稿和活动稿最后一轮校对。

痛点

AI 很会写得顺，但顺不等于可信；很多稿子的问题不是语法，而是没有人的判断、现场和边界。

怎么用
  1. 把当前稿和原始材料一起给出。
  2. 先标问题，不急着整篇重写。
  3. 逐句改掉空泛、漂亮但不稳的表达。
输入 / 输出样例

输入：AI 初稿、原始材料、编辑反馈或合作方要求。
输出：不像人的句子、问题原因、改写建议和最终可发版本。
