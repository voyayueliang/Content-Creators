  1. 01选一个最接近的任务。

先设边界，再让 AI 进入流程

Human in the loop 解决“人要参与哪一步”，Humanity in the loop 继续追问：人的判断、关系分寸、现场感和最终责任怎样被保留下来。这里的 AI 负责把材料摊开、整理缺口、生成候选，事实、角度、公开边界和最终声音都要回到创作者手里。

先问四件事

  • 01我想让 AI 帮我省哪类力？
  • 02哪些判断必须由人来做？
  • 03哪些信息不能进入公开输出？
  • 04最后这篇内容要像谁在说话？

这些工作流怎么判断能不能上架

每条工作流都要经过同一套检查：有没有真实场景、明确输入、可检查输出、月亮自己的方法、人机边界、验证等级和下一步案例。还没跑通的内容会先放进雷达候选，不急着放到公开页。

上架标准

  • 01能解决一类人的重复问题。
  • 02同时给出 prompt、判断节点和质检门。
  • 03标清 L1-L4 验证等级，不把试点包装成成熟产品。

工作流清单

每条工作流都可以轻量使用，也可以进入 GitHub 获取完整 Skill。轻量使用适合马上跑一篇稿；完整 Skill 适合放进自己的 AI 工具和内容工作台里长期复用。

Method Layer

人机协作边界与内容判断方法

这是所有工作流前面的底层方法。它帮助内容创作者在使用 AI 前先分清：哪些劳动可以交给工具，哪些判断、关系、证据和公开责任必须留在人手里。

查看完整工作流说明 
打开人机协作边界与内容判断方法。

我现在要处理的内容是：[活动稿 / 人物稿 / 口述材料 / 演讲 PPT / 作品集 / 商业合作稿 / 其他]
原始材料在这里：[粘贴材料、链接或文件路径]
发布或使用场景：[公众号 / 作品集 / 客户交付 / 内部整理 / 公开分享 / 不确定]
我最担心的问题：[事实错误 / 太像 AI / 边界不清 / 判断被写偏 / 关系风险 / 其他]

请先不要直接写正文，先帮我做一张人机协作边界卡：
1. 这次 AI 可以帮我省哪些力：整理、提取、比对、生成候选、校对、复盘。
2. 哪些判断必须留给我：角度、取舍、公开边界、关系分寸、最终声音、是否发布。
3. 哪些信息需要保护：隐私、未确认事实、合作方内部口径、未授权图片、敏感关系。
4. 哪些事实需要校对：时间、地点、身份、机构、引用、数字、链接、授权。
5. 哪些表达容易出现 AI 味：空泛总结、过度顺滑、万能转折、替读者下判断、议程排列。
6. 最后给我一个执行建议：现在适合继续写、先补材料、先做调研，还是先确认边界。

什么时候用

开始任何内容工作流前、AI 初稿让你觉得“不对劲”时、商业合作或公开发布风险较高时。

准备什么

原始材料、发布场景、读者对象、公开边界，以及你最担心 AI 写错或写偏的地方。

会得到什么
  1. AI 可代劳与人必须判断的分工。
  2. 事实、证据、关系和公开边界清单。
  3. 继续写、补材料或先确认的下一步建议。

演讲 / 口述思路 / HTML 网页 PPT

口述转 HTML 演讲 PPT

把一段口述思路，经过 AI 追问、飞书整理、图片入场和 HTML 模板生成，变成可以现场讲、可以浏览器打开的网页 PPT。

什么时候用

适合准备公开分享、工作坊、课程导入、项目路演、个人方法展示。尤其适合脑子里已经有判断，但还没有整理成演讲结构的时候。

准备什么

一段录音或转写、飞书文档、听众画像、演讲时长、想让听众记住的判断、图片/截图/logo/案例，以及喜欢的视觉方向。

会得到什么
  1. 演讲主线地图：核心问题、听众张力、主要判断、结构节奏。
  2. 飞书可继续编辑的 slide outline：屏幕内容、口头讲述、视觉素材、待确认。
  3. HTML 网页 PPT 生成 brief，以及发布前的逻辑、演讲、素材、事实、视觉和去 AI 味检查。
