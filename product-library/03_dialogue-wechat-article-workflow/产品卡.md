# 通用对谈转公众号工作流

## 状态

已打包 / 待继续验证

## 一句话

把播客、直播、访谈、圆桌逐字稿转成公众号对谈稿或内容资产。

## 目标用户

- 播客主理人
- 直播栏目运营
- 访谈内容编辑
- 创作者

## 真实痛点

对谈内容的价值常常在真实来回、犹豫、追问和关系纹理里。普通 AI 很容易把它改成一篇过度顺滑的观点文。

## 交付承诺

先判断内容形态，再输出对谈稿、对谈摘要、观察文、视频号脚本或素材卡，并保留事实、隐私和关系边界。

## 月亮的方法

这条工作流来自月亮长期处理对谈、播客、直播和圆桌材料的经验。

月亮的方法是先保护对话里的真实来回：谁在追问、谁在犹豫、哪里出现了关系和现场感。AI 先做说话人地图、议题地图和可公开边界，再判断它适合变成对谈稿、观察文、人物稿、短视频脚本还是素材卡。只有当形态判断清楚后，才进入正文。

## 工作流

1. 读取逐字稿、音视频摘要或关键片段。
2. 建立说话人地图和话题地图。
3. 标出有信息量的原话、追问、分歧、犹豫和关系线索。
4. 判断输出形态。
5. 给出结构和需要确认的事实、身份、授权边界。
6. 生成对应内容，并回收素材卡和后续选题。

## 最小交付物

- Source map
- Speaker map
- 文章形态判断
- 对谈稿 / 摘要 / 观察文
- 图片占位
- 事实与隐私检查
- 内容资产回收

## 人机边界

AI 可以做结构整理、形态判断、草稿生成和风险标注。

人必须判断对话关系如何呈现、哪些原话保留、哪些身份和细节需要匿名、是否适合公开发布。

## 验证等级

L1 到 L2 之间。通用 skill 文件夹和 README 已形成，但还需要 1-2 个公开案例证明它能稳定跑完不同类型的对谈材料。

## 质量门

- 对话门：保留真实来回，不强行改成观点文。
- 形态门：先判断输出形态，再写作。
- 关系门：隐私、身份和上下文边界清楚。
- 声音门：保留人的口气和判断，不磨成平滑总结。
