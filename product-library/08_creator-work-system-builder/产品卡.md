# 创作者作品系统搭建 Skill

## 状态

可试点 / 已有月亮个人简历、作品集和个人网站实践 / GitHub 已发布

## 一句话

从求职、申请、合作或个人品牌目标出发，把经历、作品、证据和叙事整理成外部读者能快速看懂的作品系统。

## 目标用户

- 内容创作者
- 独立工作者
- 求职者
- 申请者
- 社区运营者
- 想把项目经验整理成作品集的人

## 真实痛点

很多人的经历并不弱，但外部读者看不懂：

- 项目很多，缺少主线。
- 只写职责，没有证据。
- 作品链接散落，不能形成可信路径。
- 简历、作品集、个人网站和合作介绍各说各话。
- 自己知道项目价值，别人不知道为什么该信任你。

## 月亮的方法

这条工作流来自月亮搭建个人作品集、硅谷岗位申请材料和 AI 工作台公开页面的过程。

月亮的方法从外部读者视角开始：对方 20 秒内需要看懂你是谁、你解决什么问题、有什么证据、适合什么机会。然后把项目按能力场景重组，把证据、截图、反馈和链接放到叙事前面，让作品自己承担信任。

## 工作流

1. 明确目标：求职、申请、合作、个人网站、公开作品系统。
2. 区分读者：HR、编辑、合作方、创始人、投资人、社区伙伴。
3. 建立证据地图：作品、数据、角色、场景、关系、反馈、可核对链接。
4. 把项目按外部读者能理解的能力类型重组。
5. 提炼 20 秒内能看懂的首屏、简历或作品集结构。
6. 标出证据不足、夸大风险和读者理解障碍。
7. 再进入简历、案例页、个人网站或合作介绍文案。

## 输入材料

- 目标场景
- 目标读者
- 已有简历
- 作品链接
- 项目清单
- 截图、数据、反馈
- 想保留的个人表达
- 公开边界

## 输出物

- 外部读者视角诊断
- 作品和证据地图
- 能力分类
- 简历 / 作品集 / 个人网站信息架构
- 重点案例排序
- 页面或简历文案草稿

## 人机边界

AI 可以做：

- 重组材料
- 从外部读者角度找主线
- 标出证据缺口
- 生成页面结构和文案初稿

人必须判断：

- 目标机会是否匹配
- 哪些经历是核心身份
- 哪些项目可以公开
- 哪些表达符合自己的长期方向
- 是否接受某种外部定位

## 验证等级

L2 完整交付。Moon 已用它整理过自己的申请材料、作品集和个人网站入口。下一步要验证 L3：让另一个创作者按工作流整理出自己的作品系统草案。

## 质量门

- 读者门：20 秒内能看懂你是谁和适合什么机会。
- 证据门：每个优势都有作品、数据、反馈或链接支撑。
- 分类门：按外部读者理解方式组织，不按自我时间线堆项目。
- 边界门：不公开敏感合作信息。
- 声音门：保留个人判断，不写成求职模板腔。

## 当前风险

- 如果材料没有截图、数据或外部反馈，容易停留在自我陈述。
- 不同目标读者需要不同版本，不能一页通吃。

## 下一步验证

- 找 1-2 个创作者试跑作品系统诊断。
- 记录他们最卡的地方：材料收集、优势判断、证据排序还是页面呈现。
- 产出一版可公开的 before / after 案例。
