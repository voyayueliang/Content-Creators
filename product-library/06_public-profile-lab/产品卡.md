# 公开资料人物深度侧写工作流

## 状态

可试点 / 已有葛小川公开资料研究型人物稿案例 / 适合工作坊验证

## 一句话

把一个互联网上已有公开材料的人物，转化成有事实约束、有人类判断、有引用核查的深度人物侧写。

## 它不是什么

这不是“一键生成人物稿”的 prompt 包，也不是人物包装服务。

它是一套 Human-in-the-loop 写作协议：

- AI 负责公开资料搜集、素材索引、人类学解剖、结构推进、引用核查、去 AI 味门和信息校对门。
- 写作者负责人类判断：阅读透镜、读者画像、框架确认、素材取舍、主体边界和最终表达。
- 每一步都通过短问题把人的判断拉出来，而不是让 AI 一口气生成完整文章。

## 目标用户

- 内容创作者
- 科技媒体候选作者
- Newsletter / Substack 作者
- 播客编辑
- VC / 孵化器内容团队
- 创始人内容团队
- 社区品牌方

## 适合什么时候用

- 想写一个小有名气的人。
- 对方已有播客、访谈、演讲、公开文章或社媒长帖。
- 你想做研究型人物稿，而不是宣传稿。
- 你需要准备作品集样稿。
- 你想从公开资料里提炼一个人的判断系统、成长路径、选择方式或领域贡献。

## 不适合什么时候用

- 公开资料太少。
- 需要独家信息或一手采访才能成立。
- 想写背书稿、包装稿、攻击稿。
- 只想快速生成，不愿意参与判断。
- 人物涉及高风险隐私、争议或法律问题。

## 最小交付物

- 人物资料查看台
- 公开资料索引
- 阅读透镜选择卡
- 读者画像选择卡
- 框架确认卡
- 逐节素材取舍问题
- 引用核查表
- 去 AI 味门
- 信息校对 / 防幻觉门
- 可继续打磨的人物稿

## 核心流程

```text
人物名 / 公开素材
-> 人物资料查看台
-> 人类学解剖
-> 阅读透镜
-> 读者画像
-> 框架逐元素确认
-> 逐节起草与素材取舍
-> 引用核查
-> 去 AI 味门
-> 信息校对 / 防幻觉门
-> 交付稿与资产回收
```

## 质量门

### 事实门

- 只用素材里有的。
- 时间、职位、数据、关系和直接引语必须能溯源。
- 找不到来源就删，或标为待确认。

### 解读门

- 不推断动机。
- 不推断情感。
- 不构建人物自己没有构建的因果。
- 不把人生写成命运故事。

### 去 AI 味门

- 不从“AI 时代”“真正重要的是”这类抽象判断开头。
- 不把人物写成某个概念、趋势或成功学样本。
- 不用伪现场化表达制造细节。
- 不把还没想完的材料写得过度完整。

### 信息校对 / 防幻觉门

- 区分人物本人原话、第三方转述、媒体观察和作者判断。
- 所有动机、情绪、因果和人生主线都必须有材料支撑。
- 公开资料模式必须主动说明不是一手专访。

## 已有案例

葛小川 / AppLovin 硅谷华人 CTO 人物稿。

这个案例证明：只用公开访谈、播客、演讲和母校座谈等材料，也可以在人类判断参与下，做出一篇研究型人物稿。但它必须清楚标注为公开资料研究型写作，不是一手专访。

## 启动句

```text
打开公开资料人物深度侧写工作流。

对象：[人物姓名]
模式：[第三方无采访 / 当事人自述 / 三方协作]
已有素材：[链接 / 文件 / 暂无，请帮我搜集]
发布渠道：[公众号 / Substack / 作品集 / 机构投稿 / 不确定]
大致字数：[比如 3000-6000 字 / 先做框架]

要求：
先不要写完整长文。
请先做人物资料查看台和素材解剖，然后用短问题问我阅读透镜、读者画像和框架判断。
最终交付前请过去 AI 味门和信息校对 / 防幻觉门。
```

## 产品形态

### 模板包

适合轻量验证。交付 Claude Project Instructions、人物资料清单、判断节点卡、引用核查表和双门检查模板。

### 实战工作坊

适合第一轮对外测试。用一个公开人物现场跑流程，参与者带着自己想写的人物完成框架草案。

### 1v1 共创

适合高客单。一起完成公开资料搜集、判断访谈、框架、逐节草稿和发布前双门检查。

### 组织版工作流

适合内容团队、VC、孵化器和社区品牌方，用来记录 founder、builder、研究者、开发者或代表成员故事。

## 下一步验证

- 补一个“去 AI 味门 + 信息校对门”的案例示范。
- 开一场小型付费或邀请制工作坊。
- 记录用户最卡在哪个判断节点。
