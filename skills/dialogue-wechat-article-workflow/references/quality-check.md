# 质检表：事实、逻辑、声音、字数与边界

## 1. 事实检查

每个公开事实必须能归类：

| 类型 | 是否可用 | 处理 |
| --- | --- | --- |
| 逐字稿明确说过 | 可用 | 可改写，不改变意思 |
| 海报 / 官方物料 | 可用 | 适合人物介绍和活动信息 |
| 用户补充 | 可用 | 优先级高 |
| 编辑推断 | 谨慎 | 用弱语气 |
| AI 补出 | 不可用 | 删除或标待确认 |

不要为了让文章完整而补事实、补因果、补成绩、补情绪。

## 2. 逻辑检查

填表：

| 小节 | 材料锚点 | 这一节的功能 | 风险 |
| --- | --- | --- | --- |
| 写在前面 |  |  |  |
| 第 1 节 |  |  |  |
| 第 2 节 |  |  |  |
| 第 3 节 |  |  |  |
| 第 4 节 |  |  |  |
| 第 5 节 |  |  |  |

检查：

- 开头问题有没有被结尾回应？
- 每节是否承担不同功能？
- 有没有漂亮但不服务主线的片段？
- 小标题之间是否能读出递进？
- 是否有拼贴感？

## 3. 声音检查

高风险信号：

- 开头太像趋势报告
- 每节都给标准答案
- 嘉宾原话被改成机构口号
- 文章比原对谈更“正确”，但更没生命
- 标题像培训广告
- 结尾像宣传文案

修复：

- 加回真实场景
- 加回短对话和原话质地
- 删除过度总结
- 用发布账号真实语气重写导语
- 让案例先出现，判断后出现

## 4. 字数检查

统计口径：

- 只算 `可发布正文`
- 不算配图占位
- 不算图片链接
- 不算编辑备注
- 不算素材卡

建议：

- 短稿：2500-3500 可读字符
- 标准对谈稿：4000-6000 可读字符
- 深度长稿：6000-8000，需确认读者耐心

分节不均衡时：

- 前面太轻：补现场、原话、逻辑过渡。
- 后面太重：拆出素材卡，删重复解释。
- 整体太短：不要灌水，补材料锚点。
- 整体太长：删枝杈，保留主线。

## 5. 隐私与伦理检查

必须检查：

- 未成年人、学生作品、家庭故事
- 客户案例、合作方、商业数据
- 嘉宾产品、融资、众筹、未发布信息
- NGO、残障人士、公益对象、敏感群体
- 私聊、饭局、线下活动中的非公开内容

处理方式：

- 匿名化
- 弱化身份线索
- 改成待确认
- 只放局部图
- 不公开，回收到内部素材卡

## 6. 图片检查

每张图都问：

- 是否有授权？
- 是否有可识别的人脸、姓名、学校、机构、地址、二维码？
- 是否涉及未成年人或第三方？
- 是否能用作品局部、手部、背影、打码、重绘替代？
- 图片是否真的帮助理解，而不是装饰？

## 7. 最终发布前问题

- 这篇文章真正回答了什么？
- 它像这个发布账号会发的内容吗？
- 对谈的人还认得自己的话吗？
- 读者为什么要读到最后？
- 哪些信息还必须让人确认？

## 7.5 四道门

发布前必须过四道门：

1. 逻辑门：这篇到底在说什么？
2. 证据门：哪些判断有材料支撑？
3. 声音门：哪里不像发布账号，或者不像对谈者？
4. 关系门：有没有把别人过度公开、过度解释或当作论点材料？

任一门没过，都不要直接发布。

## 8. 人机边界检查

- AI 有没有替人确认发布、授权、署名或关系风险？
- 有没有把 `待确认` 写成确定事实？
- 有没有为了顺滑补出原材料没有的因果、情绪或成果？
- 有没有把人的“不对劲”反馈只当作普通润色意见？
- 有没有把复杂对谈压成过度完整的标准答案？
- 有没有把个人账号、机构账号、嘉宾账号的声音混在一起？
- 最终是否保留了人类发布者的确认入口？
