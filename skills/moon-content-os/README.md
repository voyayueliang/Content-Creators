# Moon Content OS Lite

一套给内容创作者、社区活动组织者、自由职业者和小型内容团队使用的轻量内容工作台。

它不是用来一次性生成一篇文章，而是帮你把零散材料、选题判断、AI 协作边界、发布前检查和发布后反馈，整理成一套可以持续复用的工作流。

## 适合谁

- 已经有不少文章、录音、活动材料、会议纪要、项目复盘，但每次写内容都要重新整理的人。
- 经常需要把活动、播客、视频、访谈、合作项目转成公开内容的人。
- 使用 AI 写作时，经常觉得初稿太顺、太空、不像自己，但又不知道怎么建立检查流程的人。
- 想把内容生产沉淀成作品集、案例、方法论、产品或服务的人。

## 第一次使用，最少准备什么

不用准备完整资料库。先准备下面三样就够：

1. 3 篇旧稿，或 1 个已经有内容材料的文件夹。
2. 1 个近期真实任务，比如一篇活动稿、播客稿、演讲稿、作品集页面或合作稿。
3. 你最在意的边界，比如事实准确、个人风格、隐私、合作方口径、读者感受。

## 复制给 AI 的启动说明

```text
打开 Moon Content OS Lite。

我想搭一个能长期复用的内容工作台，不是只写一篇文章。

我的身份或使用场景是：[内容创作者 / 社区运营者 / 活动组织者 / 自由职业者 / 个人品牌 / 小团队]
我现在最常处理的内容有：[公众号 / 播客或视频转文章 / 活动报道 / 演讲 PPT / 作品集 / 商业合作稿 / 社群内容]
我手上已有的材料包括：[文件夹路径 / 飞书文档 / 录音转写 / 会议纪要 / 旧稿 / 已发布链接 / 读者反馈 / 项目复盘]
我最想解决的问题是：[材料太散 / 不知道写什么 / AI 写得不像我 / 发布前怕出错 / 做完项目没有沉淀 / 内容不能变成产品]
我希望保留的个人判断是：[我的表达风格 / 价值立场 / 关系边界 / 事实谨慎度 / 读者感受 / 商业合作口径]

请先不要写正文。请先帮我搭一个“今天就能开始用”的最小内容工作台，输出以下内容：

1. 用户画像判断
- 判断我更像哪一类用户：深度内容创作者、社区活动组织者、自由职业者、小型内容团队，或其他。
- 说明我最应该先解决哪一个内容问题。

2. 材料地图
- 把我已有材料整理成表格：材料名称、来源、能用来做什么、缺什么、风险或边界、下一步动作。
- 如果材料不足，列出最多 5 个必须补充的问题。

3. 最小文件夹结构
- 给我一套可以直接创建的文件夹结构。
- 至少包含：素材收件箱、素材卡、选题池、发布队列、AI 协作原则、去 AI 味检查、事实校对、发布后反馈回收、产品化回收。
- 每个文件夹说明“放什么、不放什么、什么时候更新”。

4. 第一批可复制模板
- 生成 5 个模板：素材卡、选题卡、发布前校对卡、去 AI 味检查卡、发布后反馈回收卡。
- 每个模板都要能直接复制到 Markdown 或飞书文档里使用。

5. 人机协作边界
- 标出哪些工作适合交给 AI：整理、分类、提取、生成候选、比对、校对。
- 标出哪些工作必须由人决定：角度、事实确认、关系分寸、公开边界、最终声音、是否发布。

6. 第一周启动计划
- 不要给长期宏大规划。
- 请给我一个 7 天计划，每天只做一件事。
- 最后指定一篇真实内容作为试跑对象，并给出下一条可以继续发给 AI 的指令。
```

## 第一次跑完应该得到什么

- 一张材料地图：知道哪些材料能写、哪些还缺证据、哪些不能公开。
- 一套文件夹结构：知道以后新材料放哪里。
- 5 张可复制模板：素材卡、选题卡、校对卡、去 AI 味卡、反馈回收卡。
- 一张人机协作边界卡：知道 AI 能做什么，人必须判断什么。
- 一个 7 天启动计划：从一篇真实内容开始，而不是重新设计一整套系统。

## 怎么判断它跑对了

不要只看 AI 有没有写出漂亮总结。更重要的是：

- 你是否少重复整理了一次材料。
- 你是否更清楚下一篇内容该从哪里开始。
- 你是否提前看见了事实、隐私、合作方口径或读者理解风险。
- 你是否多沉淀了一张下次还能继续用的卡片。

如果这些发生了，这个工作台就开始起作用了。
