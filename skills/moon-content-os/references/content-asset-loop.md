# Content Asset Loop

Use this reference when turning a creator's workbench into a repeatable operating loop.

## Loop

```text
material intake
-> material judgment
-> content form decision
-> human-AI boundary card
-> draft or asset creation
-> fact and voice gates
-> publish or archive
-> feedback recovery
-> portfolio / product / skill recovery
```

## Material Judgment

Do not send every material into writing. Sort it first:

- `source`: useful later but not ready.
- `topic`: worth developing.
- `draft`: ready for production.
- `case`: proves an ability.
- `method`: contains reusable judgment.
- `product signal`: points to a repeatable problem or paid need.
- `private`: should not become public.

## Platform Decision

Choose output form by reader and evidence:

- WeChat article: field texture, long argument, event/story recap.
- Short social post: one observation, signal, or scene.
- Video script: one spoken point and one clear emotional line.
- Portfolio case: proof, role, process, result, what it shows.
- Internal card: useful but not public.
- Skill: repeated workflow with clear inputs, gates, and outputs.

## Recovery Rule

Every published or delivered piece should produce at least one recovered asset:

- source card
- method rule
- reusable prompt
- feedback signal
- portfolio proof
- product opportunity
- collaboration note

If nothing is recovered, the workbench is leaking value.
