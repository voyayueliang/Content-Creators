# 月亮的 AI 工作台｜工作流雷达

这个雷达用来发现哪些内容创作工作流值得进入“月亮的 AI 工作台”。

它只负责发现、评分、整理候选，不自动上架。最终是否进入公开页、GitHub Skill 或产品库，必须由月亮确认。

## 为什么需要雷达

很多真正有价值的工作流，不是一开始就被设计出来的，而是在具体项目、对话、改稿、交付和复盘里跑出来的。

雷达要做的事：

- 从聊天记录、项目复盘、草稿修改、客户交付和内容实验中发现可复用方法。
- 判断它只是一次性技巧，还是可以封装成别人也能用的 Skill。
- 把候选工作流放进人工确认队列，避免把半成品放到公开工作台。
- 保留“人来判断”的入口：成熟度、边界、命名、公开范围，都要由月亮确认。

## 雷达流程

1. **发现候选**
   - 扫描对话导出、项目文档、复盘记录、skill 草稿和产品库。
   - 标记出现“复用、产品化、模板、工作流、可上架、质量门、去 AI 味”等信号的片段。

2. **初步评分**
   - 是否有明确场景。
   - 是否有稳定输入材料。
   - 是否能产出明确结果。
   - 是否已经在真实项目中跑过。
   - 是否包含事实校对、边界检查、去 AI 味或人工确认门。

3. **进入候选池**
   - 只把“可能值得封装”的内容整理成候选卡。
   - 不把原始私密材料直接公开。

4. **月亮确认**
   - 月亮决定：不上架、继续打磨、进入试点、进入公开工作台。

5. **封装上架**
   - 通过确认后，再整理为页面卡片、GitHub Skill、产品卡或内部方法库。

## 成熟度状态

| 状态 | 含义 | 是否能公开 |
| --- | --- | --- |
| `raw` | 刚发现的片段，还没有整理 | 否 |
| `candidate` | 有可复用迹象，等待月亮判断 | 否 |
| `pilot` | 已经跑过 1-2 次，可以小范围试用 | 谨慎 |
| `live` | 验证稳定，可以放入公开工作台 | 是 |
| `hold` | 有价值，但暂时不适合产品化 | 否 |
| `reject` | 过于一次性、风险高或没有复用价值 | 否 |

## 推荐使用方式

当你在其他对话中觉得某个流程可能值得沉淀时，可以说：

```text
把这个工作流放进雷达候选池，先不要上架，等我确认。
```

或者：

```text
用工作流雷达扫一下这段对话，看看有没有适合进入月亮 AI 工作台的候选。
```

如果有导出的对话或项目文档，可以放到任意文件夹后运行：

```bash
python3 workflow-radar/scripts/scan_workflows.py <要扫描的文件夹>
```

脚本只做初筛。真正的封装和上架，要继续经过 `radar-rules.md` 里的确认规则。
