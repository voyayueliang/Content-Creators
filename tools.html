月亮的 AI 工具箱 GitHub 总仓库

Reusable AI workflows for content work

把真实材料放进去，跑出能交付、能校对、能复用的内容稿。

这是月亮沉淀出来的一组内容工作流。它不展示复杂方法论，只告诉你：适合什么场景，解决什么痛点，准备什么材料，怎么部署并开始用。

使用方式

  1. 选一个最接近的内容场景。
  2. 按说明准备材料和边界。
  3. 打开 GitHub 部署包，装进 Codex 或复制启动句试跑。
  4. 交付前过事实校对和去 AI 味两扇门。

四个可复用场景

先解决一个具体任务，再慢慢沉淀成自己的内容系统。下面每个入口都尽量保持简单：场景、痛点、怎么用，以及可部署的 GitHub 包。

活动预告 / 分论坛介绍 / 商业合作稿

社区活动内容传播工作台

把策划案、会议纪要、议程和合作方口径，整理成可审核的活动传播稿。

应用场景

社区活动预告、论坛介绍、合作方发布稿、商业合作内容交付。

痛点

活动信息很多，但传播稿容易写成资料堆砌；商业合作又不能出错，需要把事实、口径和战略价值分清楚。

怎么用

  1. 准备活动基本信息、策划案和嘉宾议程。
  2. 补充合作方希望传达的价值和不能公开的信息。
  3. 先做事实缺口检查，再进入草稿。

会后报道 / 活动复盘 / 社区记忆

活动复盘转公众号

把现场照片、群聊反馈、口述复盘、公开报道和伙伴记录，整理成会后传播包。

应用场景

活动结束后 24-72 小时内的报道、复盘、伙伴 proof 和社区记忆整理。

痛点

活动结束后材料很散，热度很快过去；如果不及时整理，现场价值和伙伴反馈都会流失。

怎么用

  1. 准备复盘口述、照片说明和公开报道链接。
  2. 标注哪些图片、名字和评价可以公开。
  3. 先生成素材地图，再判断写报道、观察稿还是传播包。

播客 / 录音 / 视频回放 / 直播

播客音视频转公众号

把音视频内容转成公众号文章、对谈稿、短观察或可继续复用的素材资产。

应用场景

播客、访谈、直播回放、圆桌逐字稿、长视频内容再分发。

痛点

音视频信息密度高，但很难直接变成可读文章；如果只做总结，又会丢掉说话人的语气、关系和现场感。

怎么用

  1. 准备逐字稿、摘要或关键片段。
  2. 说明说话人身份、发布平台和授权边界。
  3. 先判断内容形态，再决定长文、对谈稿还是短观察。

公众人物 / 创始人 / Builder Profile

公众人物深度报道工作台

基于公开访谈、播客、演讲、文章和社媒资料，写有事实约束的人物深度报道。

应用场景

公众人物、创始人、开发者、创作者、行业人物的资料研究型长文。

痛点

公开资料很多，但容易写成百科、PR 稿或主观判断；这个工作流先把事实、引用、推断和边界分开。

怎么用

  1. 给出人物姓名和公开材料链接。
  2. 说明发布渠道、读者和不能推断的边界。
  3. 先做资料查看台和文章角度选择，再写逐节草稿。

模式：[第三方无采访 / 当事人自述 / 三方协作]

发布渠道：[公众号 / Substack / 作品集 / 机构投稿 / 不确定]

请先做人物资料查看台和素材解剖，不要直接写完整长文。

最后都要过两扇门

事实校对门：名称、日期、身份、引用、授权、商业口径都要对得上材料。去 AI 味门：删掉空泛判断、硬凑转折、过度顺滑和没有来源支撑的漂亮话。
